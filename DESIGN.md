# 꿈이든 내일탐색: 통합 UX 디자인 가이드 & 사용자 여정 설계서
> **Target Audience:** 특수학급 아동 (느린 학습자, 발달장애 학생) 및 특수교육 교사  
> **Documentation Type:** Single Source of Truth (통합 디자인 시스템 & 여정 가이드라인)

본 문서는 현재 가장 높은 완성도로 구현된 **랜딩 페이지(LandingHero)**의 비주얼과 조작계 톤을 기준으로, 서비스 전반에 일관되게 흐르는 **디자인 원칙**과 **체험 시작 후 사용자 여정(User Journey) 페이지들의 개선안**을 정의한 종합 기획 설계서입니다.

---

## 1. 핵심 디자인 원칙 (Core Design Principles)

### 🎨 비주얼 시스템 (Visual Guidelines)
현재 랜딩 페이지에서 성공적으로 정립된 미학적 요소를 모든 여정 페이지로 통일하여 확장합니다.

1. **테마 맞춤형 파스텔톤 배경 (Pastel Themes):**
   * 직업별 고유 색상(바리스타: `#F2E4D6`, 도서관 사서: `#D8E5F2`, 제빵사: `#EDE3F4`)을 배경으로 깔끔하게 유지합니다.
2. **노이즈 그레인 오버레이 (Grain Texture):**
   * 랜딩 페이지의 `GRAIN_SVG` 질감을 모든 화면 최상단에 투명도 `0.4` 오버레이로 레이어링하여 아날로그하고 감성적인 통합 브랜드 아이덴티티를 유지합니다.
3. **카드 테두리 최소화 및 플로팅 레이아웃 (Cardless Floating):**
   * 불투명하고 단단한 사각형 상자(Card Box) 형태를 덜어내고, 여백과 폰트 크기 조정을 통해 텍스트와 UI가 화면 위에 자연스럽게 녹아들며 떠 있는 느낌을 구현합니다.
4. **일관된 버튼 스케일 및 호흡 애니메이션 (Tactile Buttons):**
   * 모든 주요 CTA(체험 시작하기, 다음 등)는 둥근 알약(Pill) 형태로 통일하고, 호흡하듯 천천히 커졌다 작아지는 모션(`animate-float-gentle`)과 마우스 오버 시 미세하게 위로 올라가는 반응(`translateY(-2px)`)을 주어 촉각적 재미를 제공합니다.

### ♿ 특수교육 맞춤형 접근성 원칙 (Special Ed Accessibility / UDL)
인지 및 신체 제어가 느린 아동을 고려한 필수 UI 설계 수칙입니다.

1. **정적 텍스트 제공 (Static Text Only):**
   * 서서히 글자가 써지는 타입라이터(Typewriter) 효과나 빠른 페이딩은 시독성이 떨어지는 느린 학습자에게 큰 인지 스트레스를 유발하므로 금지합니다. 모든 안내 텍스트는 **즉시 정적으로 온전하게 노출**되어야 합니다.
2. **다중 감각 지원 (TTS & Visual Redundancy):**
   * 텍스트 정보는 시각 자료(AAC 픽토그램 등)와 음성 청각 피드백(TTS/다시 듣기)을 반드시 병행합니다.
3. **충분한 터치 타깃 (Large Touch Targets):**
   * 미세 운동 조작이 서툰 학생들을 배려해 모든 인터랙션 버튼의 타깃 크기를 최소 `48px * 48px` 이상, 권장 `64px * 64px` 크기로 확보합니다.
4. **주도성 보장 (No Automatic Timer):**
   * 시간제한이나 자동 화면 넘김 기능을 절대 두지 않고, 아동이 스스로 인지하고 버튼을 눌러 다음 단계로 넘어갈 수 있도록 자율적 템포를 보장합니다.

## 2. 사용자 여정: 1단계 - 온보딩 진입로 (Onboarding)

> **여정 시점:** 랜딩 페이지에서 '체험 시작하기' 클릭 직후 ➡️ 직업 본 화면 진입 전 과도기 안내 화면

아이들의 정서적 안정을 유도하는 캐릭터 '이든'의 스토리텔링과 음성 안내가 융합된 입체적인 온보딩을 구현합니다.

```
[ 랜딩 페이지: 체험 시작 ] ──(부드러운 화면 전환)──> [ 이든의 음성 환영 카드 ] ──(디오라마 슬로우 줌인)──> [ 직업 체험 본화면 ]
```

### UI 레이아웃 설계
* **배경:** 현재 선택된 직업 테마의 파스텔 배경색이 유지되며, 디오라마 이미지는 뒷배경으로 아득하게 배치됩니다.
* **이든(Eiden) 캐릭터:** 현재 직업 유니폼(예: 바리스타 앞치마, 제빵사 모자)을 갖춰 입은 친근한 포즈의 캐릭터 이미지가 우측에 크게 나타납니다.
* **플로팅 대화 상자 (Left):**
  * Eiden의 말풍선이 좌측에 넓게 정적으로 노출됩니다.
  * **문구:** *"안녕! 나는 오늘 너와 함께할 바리스타 이든이야. 오늘 나랑 같이 향긋한 원두를 고르고 에스프레소를 내려볼까? 준비됐다면 아래 버튼을 눌러줘!"*
  * **청각 아이콘:** 말풍선 우측 상단에 눈에 띄는 **'다시 듣기(스피커)'** 아이콘을 배치하여 오디오로 말풍선 텍스트를 읽어주는 TTS 기능을 제공합니다.
* **진입 액션 버튼:**
  * 하단 중앙에 배치된 큰 알약 모양 버튼: **"준비 완료! 체험 들어가기 🚀"**

## 3. 사용자 여정: 2단계 - 직업 체험 및 여정 페이지 개선 (Subsequent Pages)

기존 구현되어 있던 투박한 UI 구성 요소들을 랜딩 페이지의 비주얼 규칙으로 세련되게 정제합니다.

### 🗺️ 직업 길찾기 화면 (JobPath)
* **현황 문제점:** 사각형의 흰색 직업 선택 카드(`.job-choice-card`)들이 지도를 많이 가려 답답해 보이며, 하단의 회색 푸터 바가 투박합니다.
* **개선 사양:**
  * 지도를 덮는 카드 배경을 투명도가 높은 유리막 형태(`rgba(255, 255, 255, 0.4)`, `backdrop-filter: blur(10px)`)로 변경하여 하단 지도 이미지가 은은하게 통과되어 보이도록 투과율을 높입니다.
  * 각 맵 노드 및 디오라마가 테두리 없이 화면 중앙 위에 입체적으로 둥둥 떠 있는 것처럼 그림자 효과(`drop-shadow`)를 강화합니다.

### 📚 직업 소개 화면 (JobIntro)
* **현황 문제점:** 직업 설명 카드와 캐릭터 카드 등이 파스텔 배경 위에서 동떨어진 흰색 사각형 상자로 존재합니다.
* **개선 사양:**
  * 모든 사각형 프레임과 배경을 제거합니다. 텍스트는 고대비 폰트로 배경에 직접 띄우고, 각 단어의 인지를 돕는 소프트 글로우(`text-shadow: 0 0 10px activeItem.bg`) 효과를 줍니다.
  * 디오라마를 에두르는 단단한 사각형 틀을 제거하고 디오라마 3D 오브젝트 자체만 자유롭게 배치합니다.

### 🎮 하루 체험 화면 (DayExperience)
* **현황 문제점:** 설명 텍스트, 이든 대화 상자, 하단 액션 바가 세로로 복잡하게 쌓여 화면 공간을 좁히고 있습니다.
* **개선 사양:**
  * 아래쪽 행동 지원 바(`.support-bar`)의 리지드한 사각형 버튼들을 랜딩 페이지 컨트롤러와 같은 **부드러운 아웃라인 형태의 둥근 알약 버튼**으로 통일합니다.
  * 이든 대화상자의 하얀 카드 배경을 제거하고, 캐릭터 일러스트와 텍스트만 떠 있는 형태로 리팩토링합니다.

### 📝 탐색 정리 & 기록 화면 (Summary & Records)
* **현황 문제점:** 소감 카드(생각 정리) 버튼 및 내 기록 앨범 카드가 다소 고전적인 사각형 그리드 레이아웃입니다.
* **개선 사양:**
  * 아동들이 직관적으로 누르기 편하도록 생각 선택 카드 리스트들을 크고 둥근 마진을 가진 카드로 다듬어 터치 실수를 방지합니다.
  * 앨범형 카드 역시 랜딩 페이지에 사용된 곡률(`border-radius: 24px`)을 상속 적용하고 마우스 오버 시 살짝 떠오르는 모션을 부여해 일관된 조작 감각을 전달합니다.

## 4. 개발 가이드라인 & 리소스 매핑

| 항목 | 타깃 대상 및 설명 | 적용 리소스 / CSS 사양 |
| :--- | :--- | :--- |
| **공통 노이즈 그레인** | 모든 화면 레이아웃 최상단 | `index.css`에 정의된 `GRAIN_SVG` 백그라운드 오버레이 적용 |
| **소프트 텍스트 글로우** | 디오라마와 겹쳐 가독성이 떨어지는 줄글 | `text-shadow: 0 0 12px activeItem.bg` 형태의 CSS 속성 바인딩 |
